### 重要教训（2025-04-11新增）
**Supabase TypeScript类型问题：**
- 问题：TypeScript严格模式下，`supabase.from('table').update(data)` 会报类型错误
- 错误信息：`Argument of type 'any' is not assignable to parameter of type 'never'`
- 根本原因：Supabase客户端的类型推断在严格模式下过于严格
- 解决方案：使用 `(supabase.from('table').update(data) as any)` 将整个链式调用断言为any
- 影响范围：所有update操作（UserDB.update, AvatarDB.update, TaskDB.update）

## 重要问题与教训记录

### ⚠️ 部署配置冲突问题（2025-04-10）

**问题描述：**
- 项目同时配置了 **Vercel** 和 **GitHub Pages** 两个部署目标
- GitHub Actions 的 `deploy.yml` 尝试部署到 GitHub Pages 时编译失败
- 导致 GitHub 持续发送部署失败通知
- 但 **Vercel 部署实际上已经成功**

**根本原因：**
- 项目初始化时同时配置了两种部署方式
- Next.js 静态导出模式与 GitHub Pages 不完全兼容
- 类型错误在 GitHub Actions 环境中无法通过编译

**解决方案：**
- 删除了 `.github/workflows/deploy.yml`
- 只保留 **Vercel** 作为唯一的部署平台
- Vercel 会自动从 GitHub 拉取代码并部署

**经验教训：**
1. **单一部署目标原则**：一个项目只配置一个主要的部署平台
2. **Vercel 优先**：Next.js 项目首选 Vercel 部署，原生支持更好
3. **及时清理配置**：切换部署平台时要删除旧的配置文件
4. **区分错误来源**：GitHub Actions 失败不等于 Vercel 部署失败

**当前状态：**
- ✅ Vercel 部署成功：https://ai-avatar-market.vercel.app
- ✅ GitHub Pages 工作流已删除
- ✅ 不再收到部署失败通知
