## 一、核心定位

### 1.1 与通用大模型的本质区别

| 维度 | 通用大模型 (ChatGPT/Claude) | 我们的AI分身 |
|------|---------------------------|-------------|
| **交互模式** | 对话式，需要反复引导 | 交付式，直接给出结果 |
| **知识来源** | 通用训练数据 | 创作者专业知识库 + 实战经验 |
| **解决问题** | 提供思路和建议 | 直接输出可执行方案/交付物 |
| **用户成本** | 高（需要懂AI、会提问） | 低（雇佣即得结果） |

### 1.2 雇佣场景的核心价值

```
用户雇佣分身的期望流程：

❌ 错误体验（当前大模型模式）：
用户: "帮我写个Python爬虫"
分身: "你需要先确定目标网站，我可以教你如何分析页面结构..."
用户: （还得自己学，反复对话）

✅ 正确体验（我们的目标）：
用户: "帮我写个Python爬虫，爬取XX网站的价格数据"
分身: （直接交付）
   ├─ 完整可运行的Python代码
   ├─ requirements.txt
   ├─ 使用说明文档
   ├─ 错误处理机制
   └─ 反爬策略建议

用户: 复制代码，配置参数，直接运行 → 拿到结果
```

### 2.2 核心模块详解

#### 模块1: 专业知识库系统 (RAG)

**目标**: 让分身拥有创作者的专业知识

**技术方案**:
```sql
-- 知识库表结构
CREATE TABLE avatar_knowledge (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    avatar_id UUID NOT NULL REFERENCES avatars(id) ON DELETE CASCADE,
    content TEXT NOT NULL,           -- 知识内容
    content_type TEXT NOT NULL,      -- 'soul', 'memory', 'document', 'code', 'solution'
    source TEXT,                     -- 来源（文件名/URL）
    embedding VECTOR(1536),          -- 向量化表示
    metadata JSONB,                  -- 元数据（标签、分类）
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 创建向量索引
CREATE INDEX ON avatar_knowledge USING ivfflat (embedding vector_cosine_ops);
```

**知识来源**:
1. **SOUL.md** - 创作者上传的人格定义文件
2. **MEMORY.md** - 创作者的经验和偏好
3. **专业文档** - PDF/Markdown/代码文件
4. **历史方案** - 过往成功任务的解决方案（自动提取）
5. **代码仓库** - GitHub/代码片段

**检索策略**:
```typescript
// 任务启动时，检索最相关的知识
async function retrieveKnowledge(avatarId: string, taskDescription: string) {
    // 1. 任务描述向量化
    const queryEmbedding = await embed(taskDescription);
    
    // 2. 检索相关知识（top-5）
    const { data: knowledge } = await db
        .from('avatar_knowledge')
        .select('*')
        .eq('avatar_id', avatarId)
        .order('embedding <-> :query', { query: queryEmbedding })
        .limit(5);
    
    // 3. 按类型组织
    return {
        soul: knowledge.filter(k => k.content_type === 'soul'),
        expertise: knowledge.filter(k => k.content_type === 'document'),
        solutions: knowledge.filter(k => k.content_type === 'solution'),
    };
}
```

// 任务完成后，自动提取经验
async function extractExperience(taskId: string) {

#### 模块3: 交付物生成引擎

**目标**: 把AI输出转化为用户可直接使用的交付物

**交付物类型**:

| 类型 | 示例 | 处理方式 |
|------|------|----------|
| **代码** | Python脚本、SQL查询 | 提取代码块 → 生成可下载文件 |
| **文档** | 方案文档、分析报告 | Markdown → PDF/HTML |
| **数据** | Excel表格、CSV | 直接生成文件 |
| **配置** | Dockerfile、YAML | 格式化后打包 |
| **多媒体** | 图片、音频 | 存储后提供链接 |

**代码提取示例**:
```typescript
// 从AI回复中提取代码块
function extractCodeBlocks(content: string): CodeBlock[] {
    const codeBlockRegex = /```(\w+)?\n([\s\S]*?)```/g;
    const blocks = [];
    let match;
    
    while ((match = codeBlockRegex.exec(content)) !== null) {
        blocks.push({
            language: match[1] || 'text',
            code: match[2].trim(),
            filename: inferFilename(match[1], match[2])
        });
    }
    
    return blocks;
}

// 生成下载包
async function createDeliverablePackage(blocks: CodeBlock[], taskId: string) {
    const zip = new JSZip();
    
    blocks.forEach(block => {
        zip.file(block.filename, block.code);
    });
    
    // 添加 README
    zip.file('README.md', generateReadme(taskId));
    
    const content = await zip.generateAsync({ type: 'blob' });
    return uploadToStorage(content, `tasks/${taskId}/deliverable.zip`);
}
```

#### 模块4: 多轮思考与任务拆解

**目标**: 复杂任务自动拆解为可执行步骤

**实现方案**:
```typescript
// 任务拆解
async function decomposeTask(taskDescription: string, avatar: Avatar) {
    const decompositionPrompt = `
        你是${avatar.name}，${avatar.description}
        
        用户任务："${taskDescription}"
        
        请将该任务拆解为可执行的子任务列表。
        每个子任务应该是：
        - 具体可执行
        - 有明确的交付物
        - 可以独立完成
        
        输出格式（JSON）：
        {
            "subtasks": [
                {
                    "id": 1,
                    "title": "子任务名称",
                    "description": "具体要做什么",
                    "deliverable": "预期交付物",
                    "estimated_tokens": 2000
                }
            ],
            "dependencies": [[1, 2]],  // 任务1完成后才能做任务2
            "total_estimate": "预计总耗时/Token数"
        }
    `;
    
    const result = await chatWithKimi(decompositionPrompt);
    return JSON.parse(result);
}

// 顺序执行子任务
async function executeSubtasks(subtasks: SubTask[], avatar: Avatar) {
    const results = [];
    
    for (const subtask of subtasks) {
        // 执行子任务
        const result = await executeTask(subtask, avatar);
        results.push(result);
        
        // 更新任务状态
        await updateSubtaskStatus(subtask.id, 'completed');
    }
    
    // 合并结果
    return mergeResults(results);
}
```
